# Product

## Register

brand

## Platform

web

## Users

맥 파워 유저와 개발자. Accessibility 권한을 요구하는 앱을 의심부터 하고, 상주 앱의 CPU·메모리·네트워크 사용을 따져보는 사람들. 랜딩 방문자와 앱 사용자는 같은 집단이며, 방문 맥락은 "⌘C가 먹었는지 몰라서 붙여넣어 확인해 본" 경험을 가진 채 해결책을 평가하는 순간이다.

주 표면은 랜딩 페이지(docs/index.html, GitHub Pages, EN/KO). 앱 자체의 UI(HUD·설정·온보딩)는 macOS 네이티브 표면으로 시스템 재질과 HIG를 따르며, 이 문서의 web 도구 범위 밖이다.

## Product Purpose

CopyNod는 ⌘C/⌘X가 실제로 성공했을 때만 체크 HUD를 띄우는 무료 오픈소스 macOS 메뉴바 유틸리티다(macOS 14+, MIT). 랜딩 페이지의 단일 목적은 방문자가 30초 안에 "이게 뭔지" 이해하고 다운로드 버튼을 누르는 것. 성공 지표도 다운로드 하나다 — 가입·뉴스레터·가격표는 존재하지 않는다.

## Positioning

키 입력은 트리거일 뿐, 클립보드 변경(changeCount)을 검증해 **실제로 복사됐을 때만** 알려준다 — 오탐 없음, 내용은 절대 읽지 않음.

## Conversion & proof

- Primary CTA: Download for macOS — 최신 릴리스 zip 직접 다운로드. Secondary: View on GitHub.
- 10초 뒤 기억할 한 줄: "복사가 실제로 됐을 때만 체크가 뜬다."
- Belief ladder (랜딩 섹션 순서가 곧 사다리): ① 이 불편은 진짜다(붙여넣어 확인하는 불안) → ② 오탐 없이 동작한다(트리거≠성공, 검증 방식) → ③ 권한을 줘도 안전하다(내용을 읽지 않음 · 수집 제로 · 오픈소스) → ④ 부담이 없다(유휴 CPU ~0% · 40MB 이하 · 무료) → 다운로드.
- Proof on hand: 아직 없음 — 실제 HUD·메뉴바 스크린샷/녹화 확보 필요(docs/assets/에는 현재 앱 아이콘뿐). 확보 시 docs/assets/ 경로로 참조한다.

## Brand Personality

조용함 · 정밀함 · 절제. 앱과 동일한 톤 — 과장 없이 사실만 짧게, 느낌표 금지. 방문자가 느껴야 할 감정은 "이 개발자는 믿을 만하다"는 안심이다. EN/KO 두 언어를 동등하게 제공하며, 한국어는 건조한 합쇼체/명사형으로 같은 결을 유지한다.

## Anti-references

- 과장 카피 — "혁신적인", "생산성 10배" 류의 하이프 언어.
- 전환 방해 요소 — 가입, 뉴스레터, 가격표.
- 과한 스크롤 애니메이션 — 히어로의 HUD 데모가 페이지의 유일한 의미 있는 모션이어야 한다.

## Design Principles

1. **주장 대신 증거** — 신뢰가 이 앱의 셀링 포인트인 만큼, 설득의 정점은 카드 속 문구가 아니라 실제 화면·공증 정보·소스 링크다.
2. **30초 페이지** — 다 읽는 데 30초. 덜어내는 쪽이 항상 옳다.
3. **제품이 곧 미학** — HUD가 보여주는 절제를 페이지가 그대로 실천한다. 조용한 제품의 랜딩은 조용해야 한다.
4. **의심 많은 방문자 기준** — Accessibility 권한을 줄지 말지 따지는 사람이 기본 페르소나. 모든 주장 옆에 검증 경로를 둔다.
5. **두 언어 동급** — EN/KO 어느 쪽도 번역투가 아니어야 하며 정보 범위도 같아야 한다.

## Accessibility & Inclusion

WCAG 2.1 AA를 목표로 한다. 본문과 작은 보조 텍스트 모두 대비 4.5:1 이상(현재 `--fg3`가 미달 — 알려진 이슈), 반복 재생되는 HUD 데모에는 prefers-reduced-motion 대안 필수. 다크/라이트 모두 지원한다.
