CopyNod 앱 아이콘 — 4가지 시안

형태·구도 비교용 플랫 시안입니다. 최종본은 macOS 26용 Icon Composer에서 Liquid Glass 레이어로 합성되며, 여기서는 각 컨셉의 실루엣과 축소 가독성(64 / 32 / 16px)을 판단하는 것이 목적입니다.

시안 A

앱이 띄우는 HUD 그 자체를 아이콘으로. 그래파이트 글래스 전면 + 흰 체크 스트로크.

  • HUD 디자인과 1:1 — 브랜드 일관성 최강
  • 16px까지 가독성 유지 (요소가 하나뿐)
  • 체크 단독이라 다른 "완료" 계열 앱과 구분이 약함

클립보드가 살짝 기울어져 "끄덕"이는 형상. 이름(Nod)의 시각화 + 그린 체크.

  • 이름과의 연결이 가장 서사적
  • 클립보드 = 클립보드 매니저로 오인될 소지 (비목표와 충돌)
  • 32px 이하에서 클립·체크 디테일이 뭉개짐

HUD 패널이 살짝 기울어 "끄덕"여주는 순간. A의 명확성 + B의 이름 연결성.

  • "HUD가 끄덕여준다" — 앱 정체성의 한 줄 요약
  • 배경/패널 2레이어 구조가 Liquid Glass 합성과 정합
  • A보다 요소가 하나 많아 16px에서 기울기 식별은 어려움
참고: 기울기(끄덕임)는 B가 −8°, D가 −6°. macOS 26에서는 시스템이 스쿼클 마스크와 글래스 재질을 강제 적용하므로, 여기의 배경 그라디언트·그림자는 근사치입니다. 시안 확정 후 레이어 분리(배경 / 패널 / 체크) → Icon Composer 합성 + 14~15용 폴백 에셋 생성 순으로 진행합니다.
