## Anti-Patterns Verdict

**AI 생성 인상: 중간 이상.** 노골적인 그라디언트 텍스트나 무분별한 글래스모피즘은 없고, 시스템 폰트·무채색·그린 체크는 제품 맥락에 맞는다. 그러나 중앙 정렬 히어로, 둥근 3단계 카드, 큰 숫자 스탯 카드, 아이콘 기능 카드, 반투명 프라이버시 패널이 이어지는 구성은 “Apple풍 유틸리티 랜딩 템플릿”으로 읽힌다. 실제 제품 장면 대신 CSS 데모와 점선 플레이스홀더가 남아 있어 CopyNod만의 고유성이 약하다.

결정적 탐지기는 경고 1개를 찾았다: `em-dash-overuse`(warning, `docs/index.html`, 집계 위치 line 0). `<title>`의 em dash까지 본문으로 계산해 임계값 5개를 넘긴 탐지기 측 false positive 성격이 강하다. 다만 실제 정적 영문 본문에도 em dash가 4회 반복되므로 문장 리듬을 다양화할 여지는 있다.

브라우저 백엔드가 없어 사용자에게 보이는 오버레이는 만들지 못했다. 대신 라이브 HTML과 로컬 `docs/index.html`의 크기 및 SHA-256이 완전히 일치함을 확인했고, 현재 배포본에도 아래 소스 기반 문제가 그대로 있음을 검증했다.

## Overall Impression

첫 5초는 좋다. “복사가 됐는지 몰라 다시 복사하거나 붙여넣어 확인한다”는 작고 정확한 불편을 즉시 붙잡고, 무료·오픈소스·macOS 14+도 빠르게 전달한다. 가장 큰 기회는 **설명 중심 페이지를 실제 제품 증거 중심 페이지로 바꾸는 것**이다. 사용자가 손쉬운 사용 권한을 허용해야 하는 제품인데, 현재 설득의 정점은 실제 화면이나 공증 정보가 아니라 카드 속 주장문이다.

## What's Working

1. **가치 제안이 즉시 이해된다.** 헤드라인, 한 문장 설명, CTA, 호환 정보의 우선순위가 명확하고 가입·가격·뉴스레터 같은 방해가 없다.
2. **프라이버시 카피가 구체적이다.** “privacy-first” 같은 추상어 대신 정수 change counter, 분석 없음, 오픈소스라는 검증 가능한 근거를 제시한다.
3. **제품과 카피 톤이 일치한다.** 과장이나 느낌표 없이 조용하고 사실적으로 말하며 영어·한국어 정보 범위도 대체로 같다.

## Priority Issues

### [P1] 다운로드 CTA의 라벨과 실제 결과가 다름

**왜 중요한가:** 페이지의 단일 성공 기준이 다운로드인데, `Download for macOS`는 ZIP이 아니라 GitHub 릴리스 페이지로 이동한다. 사용자는 다시 파일을 선택해야 하고 첫 행동부터 기대가 어긋난다.

**수정:** 최신 ZIP 직접 링크를 자동 갱신하고 CTA 옆에 `v0.1.0 · 용량 · Apple 공증 완료 · macOS 14+`를 표시한다. 직접 링크가 어렵다면 버튼 라벨을 `GitHub에서 v0.1.0 받기`처럼 정확히 바꾼다. 페이지 마지막에도 같은 CTA를 배치한다.

**추천 명령:** `$impeccable clarify docs/index.html`

### [P1] 실제 제품 증거 대신 플레이스홀더가 남아 있음

**왜 중요한가:** 설치 섹션의 점선 `Menu bar dropdown` 박스는 라이브 페이지를 미완성처럼 보이게 한다. 손쉬운 사용 권한을 요구하는 앱에서 실제 화면은 장식이 아니라 신뢰 장치다.

**수정:** 실제 Mac 화면 위에 HUD가 나타나는 짧은 캡처를 히어로 중심에 배치하고, 메뉴바 드롭다운과 권한 설정 화면을 실제 이미지로 보여준다. `Read the source`는 저장소 루트보다 클립보드를 읽지 않는 구현 파일로 깊게 연결한다.

**추천 명령:** `$impeccable bolder docs/index.html`

### [P1] 낮은 텍스트 대비와 모션 접근성 누락

**왜 중요한가:** `--fg3`는 다크 약 2.84:1, 라이트 약 2.32:1로 작은 호환 정보·단계 번호·캡션에 쓰기엔 낮다. HUD/키 애니메이션은 무한 반복되지만 `prefers-reduced-motion` 대안이 없다.

**수정:** 작은 보조 텍스트용 색을 최소 4.5:1로 올리고, 모션 감소 환경에서는 정적인 체크 상태를 표시한다. 언어 버튼에는 `aria-pressed`, FAQ에는 `aria-expanded`/`aria-controls`, 장식 기호에는 `aria-hidden`을 적용한다. 언어·푸터 터치 영역도 44×44px 수준으로 키운다.

**추천 명령:** `$impeccable audit docs/index.html`

### [P2] 작은 모바일 화면에서 레이아웃과 CTA 맥락이 맞지 않음

**왜 중요한가:** `.shot-slot`은 고정 300px인데 320px 화면의 main content 폭은 약 272px라 본문 컬럼을 넘는다. 언어 버튼도 최소 터치 크기를 보장하지 않는다. 휴대폰 사용자는 Mac용 다운로드를 바로 실행할 수 없다.

**수정:** `.shot-slot { width:min(300px,100%); }`, 그리드는 `minmax(min(100%,280px),1fr)`로 바꾸고 320/360px을 검증한다. 모바일에서는 `Mac으로 링크 보내기`, 링크 복사, QR 중 하나를 주 행동으로 제공한다.

**추천 명령:** `$impeccable adapt docs/index.html`

### [P2] 중반부가 카드 카탈로그로 평평해지고 마지막 전환을 놓침

**왜 중요한가:** Problem, How, Stats, Features가 유사한 카드/중앙정렬 문법으로 이어져 “30초 안에 읽는 페이지”보다 길게 느껴진다. FAQ 뒤 곧바로 푸터가 나와 반론이 해소된 순간 다운로드 행동으로 연결되지 않는다.

**수정:** Problem을 히어로에 통합하고, How와 실제 HUD 캡처를 하나의 증거 시퀀스로 묶는다. 기능 7개는 `오탐 없음 / 내용을 읽지 않음 / 상주 부담 없음`의 3개 차별점으로 압축하고 나머지는 FAQ로 옮긴다. 끝은 FAQ가 아니라 확신 문장과 다운로드 CTA로 마감한다.

**추천 명령:** `$impeccable distill docs/index.html`

## Persona Red Flags

### Jordan — 처음 접하는 사용자

- 다운로드 버튼이 ZIP이 아닌 GitHub 릴리스 페이지를 열어 첫 행동부터 기대와 어긋난다.
- HUD, change counter, Space, Sparkle이 설명 없이 등장한다.
- 실제 메뉴바 UI가 플레이스홀더라 설치 후 어디서 설정하는지 알기 어렵다.
- 손쉬운 사용 권한의 이유는 설명하지만 시스템 설정 경로와 공증 정보를 CTA 가까이에서 확인할 수 없다.

### Riley — 꼼꼼한 검증 사용자

- `One network call`, `Nothing else, ever`는 업데이트 다운로드·리디렉션까지 따지면 공격받기 쉬운 절대 표현이다.
- JS가 차단되거나 오류가 나면 steps/privacy/stats/features/install/FAQ가 빈 컨테이너로 남는다.
- 버전, 파일 크기, 체크섬, 공증·서명 상태를 페이지에서 확인할 수 없다.
- `Read the source`가 저장소 루트로만 연결되어 프라이버시 주장을 검증하려면 관련 파일을 직접 찾아야 한다.

### Casey — 모바일 방문자

- 휴대폰에서 주 CTA가 Mac용 다운로드뿐이고 `Mac으로 보내기` 경로가 없다.
- EN/KO 버튼과 푸터 링크의 터치 영역이 작다.
- 320px 화면에서 고정 300px 스크린샷 슬롯이 본문 폭을 넘는다.
- 긴 페이지를 다 읽은 뒤 다운로드하려면 다시 상단으로 올라가야 한다.

## Minor Observations

- 한국어 페이지에서 `Accessibility`, `Applications`, `Made by`가 남는다. `손쉬운 사용`, `응용 프로그램`, 제작자 표기 등 용어를 통일해야 한다.
- `Liquid Glass … HUD fallback`은 HUD가 없어지는 것처럼 읽힐 수 있어 더 정확한 호환 표현이 필요하다.
- FAQ 질문 6개와 기능 7개가 한 번에 같은 무게로 보여 스캔 부하가 생긴다.
- 초기 HTML이 다크 테마로 고정된 뒤 JS가 시스템 테마를 적용해 라이트 모드에서 짧은 플래시가 생길 수 있다.
- OG 이미지는 정사각형 앱 아이콘뿐이다. 실제 HUD 장면을 담은 1200×630 소셜 카드가 설명력이 높다.
- canonical URL, 지원/이슈 링크, `noscript` 대체가 없다.
- 영문 em dash 반복은 탐지기 경고 자체는 과대 계상이지만, 세 문장과 캡션의 리듬을 다르게 다듬을 가치는 있다.

## Questions to Consider

1. CopyNod는 “기능 7개짜리 메뉴바 앱”인가, 아니면 “⌘C 직후의 불안을 한 번의 체크로 없애는 앱”인가?
2. 손쉬운 사용 권한을 허용해야 한다면, 왜 가장 강한 신뢰 증거가 실제 화면과 Apple 공증 정보가 아니라 카드 속 주장문이어야 하는가?
3. 중간 카드 콘텐츠의 절반을 지워도 다운로드 결정에 필요한 정보가 남는다면, 지금의 긴 페이지는 누구를 위한 것인가?
4. 사용자가 페이지를 떠난 뒤 기억해야 할 한 장면은 추상 체크 카드인가, 실제 Mac 화면 위 CopyNod HUD인가?
